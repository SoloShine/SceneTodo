# SceneTodo - Session Archiving & Handoff Skill

> **Purpose**: 专门处理 SceneTodo 项目的会话归档和交接流程
> **Context**: 基于 `Doc/00-必读/会话归档与交接规范.md` 的标准流程
> **适用场景**: 每次开发会话结束后必须执行

---

## 📋 归档流程（5个阶段）

### 阶段 1: 会话前置验证（会话结束时执行）

**检查清单**：
- [ ] 所有计划功能 100% 完成
- [ ] 零 bug、零报错、零崩溃
- [ ] 功能验收通过
- [ ] 代码已提交（如有）

**文档准备**：
- [ ] 会话总结已创建
- [ ] 执行列表已创建
- [ ] 变更文档已创建
- [ ] 关键注释已添加

**下一会话准备**：
- [ ] 下一会话目标已明确
- [ ] 遗留问题已列出
- [ ] 时间预估已完成

---

### 阶段 2: 会话记录归档（执行中并行）

#### 目录合并（如有多部分目录）

```powershell
# 移动文件到统一目录
Move-Item "Session-N-PartX/*.md" "Session-N/"
Remove-Item "Session-N-PartX" -Recurse
```

**合并验证**：
- 更新 `Session-N-归档完成报告.md`
- 记录合并文件清单
- 说明合并原因

#### 更新会话 README

**必备内容**：
- [ ] 更新文档目录
- [ ] 更新统计信息
- [ ] 添加归档说明

**参考模板**：
```markdown
### 📁 文档结构
### 归档文档
#### N. [Session-N-归档完成报告.md] 🔒
**归档说明** - 归档流程和总结
```

---

### 阶段 3: 规划文档归档（执行中并行）

#### 识别待归档文件

**待归档清单**：
```
06-规划文档/
├── README-Session-N-规划说明.md         [必须归档]
├── Session-N-1至Session-N-衔接文档.md   [必须归档]
├── Session-N-执行规划.md               [必须归档]
├── Session-N-执行清单.md               [必须归档]
```

#### 执行列表归档

```powershell
# 移动规划文档到会话记录
Move-Item "06-规划文档/README-Session-N-规划说明.md" `
          "01-会话记录/Session-N/"
Move-Item "06-规划文档/Session-N-1至Session-N-衔接文档.md" `
          "01-会话记录/Session-N/"
Move-Item "06-规划文档/Session-N-执行规划.md" `
          "01-会话记录/Session-N/"
Move-Item "06-规划文档/Session-N-执行清单.md" `
          "01-会话记录/Session-N/"
```

#### 更新规划目录说明

**文件**: `06-规划文档/README-规划文档说明.md`

**更新内容**：
- [ ] 更新当前规划为 Session-N+1
- [ ] 更新当前目录结构
- [ ] 添加 Session-N 归档记录
- [ ] 更新归档历史列表

**参考模板**：
```markdown
> **最后更新**: YYYY-MM-DD
> **当前规划**: Session-N+1

### Session-N 规划文档已归档 🔒
**归档位置**: `Doc/01-会话记录/Session-N/`
**归档日期**: YYYY-MM-DD

## 📝 归档历史
| 会话 | 归档日期 | 归档文件数 | 归档位置 |
|------|---------|-----------|---------|
| Session-N | YYYY-MM-DD | 4个 | `01-会话记录/Session-N/` |
```

#### 更新会话 README

**添加归档文档的链接**：
```markdown
### 📁 归档规划文档（已归档）

#### N. [README-Session-N-规划说明.md] 📋
**规划说明** - Session-N规划文档说明

#### N+1. [Session-N-1至Session-N-衔接文档.md] 🔄
**衔接文档** - Session-N-1至Session-N的衔接
```

#### 创建归档清单

**文件**: `Session-N-规划文档归档完成清单.md`

**必备内容**：
- 归档日期
- 归档人员
- 文件清单
- 核对验证
- 归档统计

---

### 阶段 4: 衔接文档编写（执行中并行）

#### PRD 进度更新

**更新内容**：
- [ ] P0 核心功能进度
- [ ] P1 优先功能进度
- [ ] P2/P3 功能状态
- [ ] 各完成百分比

**更新格式**：
```markdown
### P0 功能：已全部实现（100%）
完成度：██████████████████████ 100% (X/Y)

### P1 功能：部分实现（XX%）
完成度：████████░░░░░░░░░░░░  XX% (X/Y)
```

#### 下一会话规划

**规划内容**：
- [ ] Session-N 成功总结
- [ ] Session-N+1 目标设定
- [ ] 推荐实施选项（3个）
- [ ] 预估时长
- [ ] 优先级排序

#### 编写衔接文档

**文件**: `06-规划文档/Session-N至Session-N+1-衔接文档.md`

**文档结构**：
```markdown
# 📋 Session-N 至 Session-N+1 衔接文档

## 📊 Session-N 总结
- 完成成果
- 主要成就

## 📊 项目当前状态
- 整体完成度
- P0/P1/P2 进度

## 🎯 Session-N+1 执行规划
- 核心目标
- 优先级排序

## 📚 必读文档
- 优先级阅读
- 阅读时间

## 🤝 交接准备
- 遗留问题
- 推荐选项

## 💡 建议实施模式
- 团队协作
- 推荐选项

## 📅 时间规划
- 开发任务
- 时间预估

## 🎯 开发提示
- 开发前检查
- 开发建议
```

## ❓ 常见问题

### Q1: 什么时候触发归档？
**A**: 会话 100% 完成、下一会话规划文档已创建、有新需求时立即归档

### Q2: 规划文档需要全部归档吗？
**A**: 是的。当前会话的衔接文档、执行规划、执行清单等相关规划文档都要归档

### Q3: 归档清单需要多详细？
**A**: 参考我们的规范，要包含详细的归档流程、总结和核对

### Q4: 目录结构需要合并吗？
**A**: 参考"目录结构规范"部分，有分散目录的文档应合并到会话目录

### Q5: 如何验证归档完成？
**A**: 使用归档完成验证清单，检查文件完整性、目录结构、文档内容、核对验证

### Q6: 衔接文档应该包含哪些内容？
**A**: Session-N 成果总结、项目当前状态、Session-N+1 执行规划、必读文档、交接准备等

### Q7: 项目文档需要同步更新吗？
**A**: 是的。参考"项目文档更新规范"，用新的功能和进度更新整体状态

### Q8: 归档流程需要多长时间？
**A**: 完整的归档流程预估 60-90 分钟，包含各阶段的工作量

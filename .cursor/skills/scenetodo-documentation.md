- ✅ **已完成** - 功能已实现并测试通过

- ❌ **未实现** - 尚未开发

## 🔄 文档更新流程

### 新功能开发时

1. **创建会话文档**（在 `01-会话记录/Session-N/` 中）
   - README.md - 会话导航
   - Session-N-会话总结.md - 会话总结
   - Session-N-执行完成清单.md - 执行列表

2. **创建规划文档**（在 `06-规划文档/` 中）
   - Session-N-1至Session-N-衔接文档.md - 衔接文档
   - Session-N-执行规划.md - 执行规划
   - Session-N-执行清单.md - 执行清单

3. **功能实现完成后更新文档**
   - 更新 `00-必读/项目状态总览.md`
   - 更新 `06-规划文档/开发路线图-v1.0.md`
   - 如需要，创建功能文档到 `02-功能文档/`

### 会话结束时

**必须执行归档流程**（5 个阶段）：
1. 会话前置验证
2. 会话记录归档
3. 规划文档归档
4. 衔接文档编写
5. 项目文档更新

详见 [scenetodo-session-archive.md](./scenetodo-session-archive.md)

## 📊 文档统计数据

### 文档行数标准

| 等级 | 行数 | 颜色 | 状态 | 建议 |
|------|------|------|------|------|
| 安全 | < 600 | 🟢 | ✅ 安全 | 无需分割 |
| 警告 | 600-800 | 🟡 | ⚠️ 警告 | 可选分割 |
| 危险 | 800-1000 | 🟠 | ⚠️ 危险 | 应当分割 |
| 严重 | > 1000 | 🔴 | ❌ 严重 | 必须分割 |

### 文档分割规则

当文档超过 1000 行时，必须按照以下规则分割：

1. **按章节分割**: 将相关章节合并到一个文件
2. **保持连贯**: 确保分割后的文档逻辑连贯
3. **添加导航**: 在每个 Part 文件中添加导航链接

**分割示例**：
```
原文档: 文档编写规范.md (1200 行)
分割后:
  ├─ 文档编写规范-Part1-基础规范.md (300 行)
  ├─ 文档编写规范-Part2-逻辑规范.md (400 行)
  ├─ 文档编写规范-Part3-归档模板.md (300 行)
  └─ 文档编写规范-Part4-FAQ及执行.md (200 行)
```

## 🎨 文档格式规范

### Markdown 格式

#### 标题层级

```markdown
# H1 - 文档标题（仅一个）
## H2 - 主要章节
### H3 - 子章节
#### H4 - 详细说明
```

#### 列表

```markdown
- 无序列表项
  - 嵌套列表项

1. 有序列表项
   1. 嵌套有序列表
```

#### 代码块

```csharp
// 代码块必须指定语言
public class Example
{
    // ...
}
```

#### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

#### 引用

```markdown
> 引用文本
```

#### 链接和图片

```markdown
[链接文本](链接地址)

![图片描述](图片地址)
```

#### Emoji 使用

使用 Emoji 增强可读性：
- ✅ 已完成
- 🚧 进行中
- ⏳ 已规划
- ❌ 未实现
- 🔥 重要
- ⚠️ 警告
- 💡 提示
- 📝 说明
- 📚 文档
- 🎯 目标
- 🔧 技术细节
- 📊 统计数据

## 📋 文档内容编写指南

### 清晰性

- **使用简单语言**: 避免复杂的术语，必要时提供解释
- **结构清晰**: 使用标题、列表、表格组织内容
- **分段合理**: 每段不超过 5 行

### 准确性

- **技术准确**: 确保代码示例、技术描述准确无误
- **更新及时**: 及时更新文档，保持与代码同步
- **验证通过**: 编写文档前验证内容准确性

### 完整性

- **前置条件**: 列出阅读本文档的前置条件
- **步骤详细**: 提供详细的步骤说明
- **示例完整**: 提供完整的代码示例

### 一致性

- **命名一致**: 使用统一的命名规范
- **格式一致**: 保持格式风格一致
- **术语一致**: 使用统一的术语

## 🛠️ 文档工具

### PowerShell 脚本

#### 文档长度检查

```powershell
.\Doc\Check-DocumentLength.ps1
```

检查所有 Markdown 文档的长度，超过 1000 行的文档会标记为严重。

#### 文档分割

```powershell
.\Doc\Tools\Split-Document.ps1
```

将过长的文档分割为多个 Part 文件。

#### 会话文档分割

```powershell
.\Doc\Tools\Split-HandoffDocument.ps1
```

专门用于分割会话归档文档。

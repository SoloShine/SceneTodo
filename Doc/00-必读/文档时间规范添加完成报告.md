## ?? 常见问题

### Q1: 如何处理历史文档的时间？
**A**: 
1. 使用Git历史查找实际时间：
   ```bash
   git log --format=%aI -- 文件路径
   ```
2. 或标注为"[估算]"：
   ```markdown
   **创建日期**: 2026-01-01 10:00:00 [估算]
   ```

### Q2: 跨时区团队如何处理时间？
**A**: 
1. 统一使用北京时间（UTC+8）
2. 在文档中明确标注时区：
   ```markdown
   **创建日期**: 2026-01-02 18:19:30 (UTC+8)
   ```

### Q3: 文档更新时必须更新时间吗？
**A**: 
- 修复错别字、格式调整：可不更新
- 新增内容、修改逻辑：必须更新
- 重大重构：必须更新并升级版本号

### Q4: 如何验证时间是否合理？
**A**: 
检查：
- 创建日期 ≤ 最后更新日期
- 同一文档内时间递增
- 相邻变更时间间隔合理（不是几秒钟）

## 13. 常见问题

### Q1: 如何快速获取当前时间？
**A**: 在终端执行 `Get-Date -Format "yyyy-MM-dd HH:mm:ss"`（PowerShell）

### Q2: 文档放在哪个目录？
**A**: 参考第7章"文档分类与存放"，根据文档类型选择目录：
- 核心文档 → 00-必读/
- 会话记录 → 01-会话记录/
- 功能文档 → 02-功能文档/
- 测试文档 → 03-测试文档/
- 技术文档 → 04-技术文档/
- 用户文档 → 05-用户文档/
- 规划文档 → 06-规划文档/

### Q3: 如何处理旧文档的时间占位符？
**A**: 可以使用Git历史查找实际时间，或标注为"[估算]"。

```bash
# 查看文件的创建时间
git log --diff-filter=A --follow --format=%aI -- 文件路径

# 查看文件的最后修改时间
git log -1 --format=%aI -- 文件路径
```

### Q4: 是否需要每次小修改都更新版本号？
**A**: 
- 修复错别字、格式调整：不需要更新版本号
- 新增章节、修改内容：更新次版本（如1.0→1.1）
- 重大重构：更新主版本（如1.X→2.0）

### Q5: 什么时候需要归档文档？
**A**: 当一个会话100%完成，且下一会话规划已就绪时，立即归档该会话的规划文档。

### Q6: 交接文档需要归档吗？
**A**: 交接文档临时保留在规划目录，方便下一轮开发查阅。当下一会话完成后，交接文档随该会话一起归档。

### Q7: 如何判断哪些文档需要归档？
**A**: 所有以"Session-XX"开头的规划文档都需要归档，但交接文档（Session-XX到Session-XX+1）临时保留。

### Q8: 归档后发现遗漏文档怎么办？
**A**: 立即补充移动遗漏文档，更新索引，并在归档说明中记录补充操作。

### 14.2 建议性规范（推荐遵守）

以下规范建议遵守，但不强制：

- 使用清晰的标题结构
- 添加必要的代码注释
- 提供示例和截图
- 定期整理文档结构
- 使用表情符号增强可读性
- 添加导航链接

### 14.3 违反规范的处理

如果发现文档违反规范：

#### 步骤1: 标记问题
在文档顶部添加 `[不规范]` 标记：
```markdown
# [不规范] 文档标题

> ?? **规范问题**: 时间使用了占位符
```

#### 步骤2: 说明具体违规内容
```markdown
## ?? 规范问题说明

本文档存在以下规范问题：
1. 使用了时间占位符（2025-01-XX）
2. 缺少版本历史表格
3. 文档超过1000行未分割
```

#### 步骤3: 提供改进建议
```markdown
## ?? 改进建议

1. 使用 `Get-Date` 获取真实时间
2. 在文档末尾添加版本历史表格
3. 按照第10.4节分割文档
```

#### 步骤4: 在下次更新时修正
- 设置提醒或TODO
- 在下次编辑该文档时优先修正
- 修正后移除 `[不规范]` 标记

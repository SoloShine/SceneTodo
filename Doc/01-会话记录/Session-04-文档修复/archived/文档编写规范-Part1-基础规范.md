## 1. 文档命名规范

### 1.1 文件命名
- **使用中文命名**（便于阅读）
- **使用连字符分隔**（多个词语）
- **禁止使用空格**（避免路径问题）

**命名格式**:
```
- 功能文档: 功能名称.md
- 测试文档: 测试指南-功能名称.md
- 会话文档: 会话交接文档-vX.X.md
- 技术文档: 技术主题.md
```

**示例**:
```
? 正确:
- 优先级功能.md
- 测试指南-右键菜单.md
- 会话交接文档-v3.0.md

? 错误:
- priority feature.md (使用空格)
- test guide right menu.md (英文且使用空格)
```

## 2. 文档结构规范

### 2.1 文档头部
每个文档必须包含以下头部信息：

```markdown
# ?? 文档标题

> **创建日期**: YYYY-MM-DD HH:mm:ss  
> **最后更新**: YYYY-MM-DD HH:mm:ss  
> **版本**: X.X  
> **状态**: [草稿/审核中/正式发布/已归档]

---

## ?? 文档说明

简要说明本文档的目的、受众和使用场景。
```

### 2.2 章节结构
- 使用标准Markdown标题层级（# ## ### ####）
- 每个一级标题前后空一行
- 使用清晰的编号和层级

### 2.3 状态标识
使用统一的状态标识符：
- ? **已完成** - 功能已实现并测试
- ? **进行中** - 正在开发或测试
- ?? **部分完成** - 部分实现，有限制
- ? **未实现** - 尚未开发
- ??? **已归档** - 旧版本，仅供参考

## 3. 时间处理规范 ??????

**这是强制性规范，必须严格遵守！**

### 3.1 时区处理
- ? **必须使用本地时区时间**记录所有日志和文档
- ? **禁止使用 UTC 或其他时区**
- ? **禁止使用占位符**（如 2025-01-XX）
- ? **时间格式**: `YYYY-MM-DD HH:mm:ss`（24小时制）

### 3.2 时间获取方法

#### PowerShell（推荐）
```powershell
Get-Date -Format "yyyy-MM-dd HH:mm:ss"
# 输出示例: 2026-01-02 22:04:47
```

#### CMD（Windows）
```cmd
date /T && time /T
```

#### Bash（Linux/Mac）
```bash
date '+%Y-%m-%d %H:%M:%S'
```

### 3.3 时间准确性要求

#### 严格要求
- ? **必须使用系统当前时间**，禁止手动设置或估算时间
- ? **精确到秒**：所有时间记录必须精确到秒级别
- ? **时间一致性**：同一文档内的时间必须按顺序递增
- ? **禁止伪造**：不得伪造或篡改时间戳
- ? **检查机制**：每次写入时间前必须重新获取当前时间

## 4. 文档版本管理

### 4.1 版本号规则
使用语义化版本号：`主版本.次版本.修订版本`

- **主版本**：重大变更，不兼容的修改
- **次版本**：新增功能，向后兼容
- **修订版本**：Bug修复，小改动

**示例**:
- v1.0 - 初始版本
- v1.1 - 添加新章节
- v1.1.1 - 修复错别字
- v2.0 - 重构整个文档

### 4.2 版本历史
在文档末尾添加版本历史表格。

## 5. Markdown 格式规范

### 5.1 标题
- 使用 ATX 风格标题（# ## ###）
- 一级标题使用表情符号增强可读性
- 标题前后空一行

### 5.2 列表
- 使用 `-` 作为无序列表标记
- 有序列表使用数字 + `.`

### 5.3 代码块
- 使用三个反引号标记代码块
- 指定语言类型（syntax highlighting）

### 5.4 链接
- 使用相对路径链接项目内文档
- 使用绝对URL链接外部资源

### 5.5 表格
- 使用管道符 `|` 分隔列
- 表头和内容之间使用 `-` 分隔

## ?? 使用建议

### 新建文档时
1. 复制文档头部模板（第2章）
2. 执行 `Get-Date` 获取当前时间
3. 填写文档元数据
4. 按规范命名文件（第1章）
5. 使用Markdown格式编写（第5章）

### 更新文档时
1. 再次执行 `Get-Date` 获取更新时间
2. 更新"最后更新"字段
3. 添加版本历史记录（第4章）
4. 必要时更新版本号

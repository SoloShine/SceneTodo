国际化功能已成功集成到 SceneTodo 项目中！

## ?? 包含内容

### 1. 核心组件
- ? `LocalizationService` - 国际化服务（单例模式）
- ? `SupportedLanguage` - 支持的语言枚举
- ? `LanguageSettings` - 语言设置模型
- ? `LanguageSettingsWindow` - 语言设置UI窗口

### 2. 资源文件
- ? `Resources/Strings.resx` - 中文资源（默认）
- ? `Resources/Strings.en.resx` - 英文资源

### 3. 集成
- ? 已集成到 `AppSettings`
- ? 已在 `App.xaml.cs` 中初始化
- ? 支持自动检测系统语言

## ?? 立即使用

### 方法1：在 XAML 中使用（推荐）

```xaml
<Window xmlns:services="clr-namespace:SceneTodo.Services">
    <!-- 使用 Localization 扩展 -->
    <Button Content="{services:Localization Common_Save}" />
    <TextBlock Text="{services:Localization Todo_Title}" />
    
    <!-- 或使用 Binding -->
    <Label Content="{Binding Source={x:Static services:LocalizationService.Instance}, Path=[Common_OK]}" />
</Window>
```

### 方法2：在 C# 代码中使用

```csharp
using SceneTodo.Services;
using SceneTodo.Resources;

// 使用 Strings 辅助类（有智能提示）
string saveText = Strings.Common_Save;
string cancelText = Strings.Common_Cancel;

// 或直接使用服务
string text = LocalizationService.Instance["Common_Delete"];
string formatted = LocalizationService.Instance.GetString("Welcome_{0}", userName);
```

### 方法3：切换语言

```csharp
using SceneTodo.Models;
using SceneTodo.Services;

// 切换到英文
LocalizationService.Instance.ChangeLanguage(SupportedLanguage.English);

// 切换到中文
LocalizationService.Instance.ChangeLanguage(SupportedLanguage.ChineseSimplified);

// 自动检测系统语言
LocalizationService.Instance.AutoDetectLanguage();
```

### 方法4：打开语言设置窗口

```csharp
using SceneTodo.Models;
using SceneTodo.Views;

private void OpenLanguageSettings()
{
    var settings = AppSettings.Load();
    var window = new LanguageSettingsWindow(settings);
    
    if (window.ShowDialog() == true)
    {
        // 用户保存了设置
        MessageBox.Show(
            LocalizationService.Instance["Message_SaveSuccess"],
            LocalizationService.Instance["Message_Info"],
            MessageBoxButton.OK);
    }
}
```

## ? 添加新的本地化字符串

### 步骤：

1. **在 Visual Studio 中打开资源文件**
   - 打开 `Resources/Strings.resx`（中文）
   - 添加新的键值对，例如：
     - 名称: `Button_Submit`
     - 值: `提交`

2. **在英文资源文件中添加对应翻译**
   - 打开 `Resources/Strings.en.resx`
   - 添加相同的键，不同的值：
     - 名称: `Button_Submit`
     - 值: `Submit`

3. **在代码中使用**
   ```xaml
   <Button Content="{services:Localization Button_Submit}" />
   ```
   或
   ```csharp
   string text = LocalizationService.Instance["Button_Submit"];
   ```

4. **（可选）更新 Strings.Designer.cs**
   在 `Strings.Designer.cs` 中添加静态属性以获得智能提示：
   ```csharp
   public static string Button_Submit => L["Button_Submit"];
   ```

## ?? 最佳实践

1. **所有用户可见的文本都应该使用本地化**
   ? 错误: `<Button Content="保存" />`
   ? 正确: `<Button Content="{services:Localization Common_Save}" />`

2. **使用有意义的资源键名称**
   ? 错误: `String1`, `Text2`
   ? 正确: `Button_Save`, `Label_UserName`

3. **保持命名规范一致**
   - `Common_*` - 通用操作
   - `Todo_*` - 待办相关
   - `Settings_*` - 设置相关
   - `Message_*` - 消息提示

4. **中英文资源键必须一致**
   确保 `Strings.resx` 和 `Strings.en.resx` 包含相同的键

## ? 常见问题

### Q: 为什么切换语言后需要重启？
A: 某些静态资源在应用启动时加载，需要重启才能完全更新。未来可以优化为动态切换。

### Q: 如何添加新语言（如日文）？
A: 1) 在 `SupportedLanguage` 枚举中添加 2) 创建 `Strings.ja.resx` 3) 更新 `LocalizationService`

### Q: 资源文件修改后是否需要重新编译？
A: 是的，资源文件会被编译到程序集中，修改后需要重新构建项目。

### Q: 如何处理带参数的字符串？
A: 使用 `GetString` 方法：
```csharp
string msg = LocalizationService.Instance.GetString("Welcome_{0}", userName);
```
